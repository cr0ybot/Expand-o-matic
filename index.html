The Extra-ordinary Expand-o-matic

Expand-o-matic is based on two wonderful CSS responsive grid systems, namely Skeleton (getskeleton.com) and the mobile-first version of Foundation (github.com/adamfairhead/mobile-first-foundation). It is also much simpler (or more basic, if you prefer) and consequentially may have less of a learning curve. Based on a mobile-first responsive grid, you should be able to quickly develop websites that look beautiful at any size and on any device. Resize this basic page to see the grid in action.

Fork Expand-o-matic on GitHub

Expand-o-matic is open open source and hosted on GitHub at github.com/cr0ybot/Expand-o-matic. Please don't hesitate to open issues if there is a problem and/or make pull requests.

Three Core Principles

Expand-o-matic borrows (with modification) the three core principles of the Skeleton framework:

  • Mobile-First Responsive Grid: Elegant scaling from mobile to tablets to desktop browsers.
  • Fast to Start: A tool for rapid development with best practices.
  • Style Agnostic: Provide basic, beautiful styles meant to be customized.

Three Core CSS Files

Expand-o-matic is composed of 3 CSS files, separated by function:

  • base.css: Includes Normalize.css and basic styles to be included on all pages, including typography, forms, and buttons.
  • grid.css: Mobile-first media queries ensure that mobile devices aren't loading unnecessary CSS data. Base styles apply to all devices and are overridden at successive device-width breakpoints to create the grid.
  • styles.css: Meant to be filled in with your own styles. Breakpoints for "standard" device-widths are provided as a starting point; you are encouraged to create your own breakpoints in-between at widths where your design breaks.
